# SiteZip 产品设计文档

## 一、产品定位

SiteZip 是一款面向开发者、设计师与研究人员的浏览器扩展。它能够将当前浏览网站在浏览器「源代码 / 来源」(DevTools → Sources) 面板下加载的全部静态资源，一键抓取、还原目录结构并打包为 ZIP 压缩包下载到本地。

一句话概括：**把 DevTools Sources 面板里那棵资源树，一键变成本地的 ZIP。**

## 二、目标用户

| 用户 | 核心诉求 |
| --- | --- |
| 前端工程师 | 快速留存线上页面的静态资源用于学习、比对、离线调试 |
| UI / 视觉设计师 | 批量获取站点图片、图标、字体等设计素材 |
| 技术研究者 | 归档网页资源用于分析与备份 |
| 教学 / 教程作者 | 制作离线演示素材 |

## 三、核心功能

1. **一键打包下载**：点击扩展图标 → 一个按钮完成抓取与下载。
2. **资源树抓取（核心）**：基于 `chrome.debugger` 的 `Page.getResourceTree`，枚举与 Sources 面板完全一致的已加载资源（HTML / CSS / JS / 图片 / 字体 / JSON 等）。
3. **DOM 解析补全**：注入脚本解析渲染后的 DOM，补全 `img / script / link / source / 内联 style url()` 等引用但可能未进入资源树的资源。
4. **渲染 HTML 留存**：可选保存渲染后的最终 HTML（`outerHTML`）。
5. **目录结构还原**：按资源 URL 的 host + pathname 还原原始目录层级；带查询串的资源自动追加短哈希避免覆盖。
6. **实时进度**：弹窗内展示进度条与操作日志。

## 四、交互设计

- **入口**：浏览器工具栏扩展图标 → 弹出 360px 宽的弹窗。
- **弹窗结构**：品牌区 → 当前站点卡片 → 抓取选项（三个可勾选项）→ 主行动按钮 → 进度条 → 日志区。
- **反馈**：每一步都有日志行；成功（绿）、警告（黄）、错误（红）分色显示。
- **降级**：非 http/https 页面禁用按钮并提示；调试器不可用时自动降级为纯 DOM 抓取。

## 五、视觉设计

采用 **Swiss / 高对比** 设计语言（Archetype 4），营造专业、克制的开发者工具气质。

| 设计令牌 | 值 |
| --- | --- |
| 背景 | `#09090B` |
| 表面 | `#18181B` |
| 描边 | `#27272A` |
| 主文字 | `#FAFAFA` |
| 次文字 | `#A1A1AA` |
| 强调色 | `#EAB308`（琥珀黄） |
| 强调悬停 | `#FACC15` |

- **字体**：标题 Cabinet Grotesk（font-black，紧字距）；正文 IBM Plex Sans；等宽 JetBrains Mono。
- **布局**：Bento 网格 + 大留白 + 非对称栅格。
- **表面**：扁平化 + 1px 描边，不使用柔和阴影。
- **按钮**：高对比、直角（rounded-none / rounded-sm）。
- **动效**：轻微位移 (`-translate-y-1`)、高对比文字色变、入场错峰渐显。

## 六、落地页设计

- **顶部导航**：左侧品牌。
- **Hero**：非对称、文字主导的首屏，暗色代码矩阵背景，主副 CTA。
- **功能区**：Bento 盒子网格，突出「资源树抓取 / DOM 补全 / 结构还原」。
- **使用步骤**：编号步骤流。
- **安装区**：下载扩展 ZIP + 安装指引。
- **免责声明区**：明确合规使用边界。
- **页脚**：文档链接与版权。

## 七、隐私与安全

- 扩展完全在**本地**运行，不上传任何抓取数据到服务器。
- 使用 `chrome.debugger` 会触发浏览器顶部「正在调试此浏览器」提示，属正常现象，打包完成后自动断开。
