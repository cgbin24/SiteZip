# 常见问题 FAQ

### Q1. 为什么顶部出现「正在调试此浏览器」提示？
这是 Chrome 使用调试协议 (`chrome.debugger`) 时的正常安全提示。SiteZip 借助它读取资源树，打包完成后会自动断开，提示随之消失。

### Q2. 抓取到的资源不全？
资源树只包含**页面已加载**的资源。请先滚动页面、触发懒加载或交互，让资源加载进「源代码/来源」面板后再打包。

### Q3. 跨域资源抓不到？
资源树阶段通常已覆盖大部分资源。DOM 补全阶段的 `fetch` 会受同源/CORS 限制，未开启 CORS 的第三方资源会被安全跳过。

### Q4. Firefox 能用吗？
可以，但 Firefox 不支持 `chrome.debugger`，会自动降级为「DOM 解析 + fetch」模式，抓取范围以页面可访问资源为准。

### Q5. 数据会上传到服务器吗？
不会。扩展完全在本地运行，所有抓取与打包均在浏览器内完成，不上传任何数据。

### Q6. ZIP 里的目录结构是怎样的？
按资源 URL 的域名与路径还原，例如 `example.com/images/logo.png`；渲染后的 HTML 存放在 `_rendered/` 目录。
