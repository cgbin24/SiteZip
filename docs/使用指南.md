# SiteZip 使用指南

## 一、安装浏览器扩展

### Chrome / Edge（推荐）

1. 从落地页点击「下载扩展」获取 `sitezip-extension.zip`，解压到本地任意目录。
2. 打开 `chrome://extensions`（Edge 为 `edge://extensions`）。
3. 打开右上角 **开发者模式**。
4. 点击 **加载已解压的扩展程序**，选择解压后的 `sitezip-extension` 文件夹。
5. 扩展出现在工具栏，建议点击图钉固定。

### Firefox

1. 打开 `about:debugging#/runtime/this-firefox`。
2. 点击 **临时载入附加组件**，选择扩展目录中的 `manifest.json`。
3. Firefox 不支持 `chrome.debugger`，将自动使用「DOM 解析 + fetch」降级模式。

## 二、抓取并打包站点资源（两步式）

1. 打开任意需要抓取的网站（http / https 页面）。
2. **建议先滚动/浏览页面**，让更多静态资源加载进「源代码/来源」面板。
3. 点击工具栏的 SiteZip 图标打开弹窗。
4. 按需勾选抓取选项，然后点击 **扫描资源列表**：
   - ☑ 抓取「源代码/来源」已加载资源（**核心**）
   - ☑ 解析 DOM 补全引用的静态资源
   - ☑ 额外保存渲染后的 HTML
5. 扫描完成后展开完整资源清单：
   - 顶部按 **类型** 过滤：HTML / CSS / JS / 图片 / 字体 / JSON / 其他，点击彩色标签即可切换启用。
   - 顶部按 **域名** 过滤：主域名置顶（带黄色左边框），第三方 CDN / 插件资源可一键排除。
   - 支持关键字搜索 URL，以及 **全选 / 取消全选 / 反选 / 仅主域名** 快捷按钮。
   - 每行显示彩色类型徽章、路径与来源（`树` = 资源树，`DOM` = DOM 补全）。
6. 勾选完成后点击 **打包所选并下载 ZIP**。
7. 顶部出现「正在调试此浏览器」提示属正常现象；等待进度条完成。
8. ZIP 自动下载到默认下载目录，文件名形如 `sitezip_example.com_20260101_120000.zip`。
9. 如需重新扫描（页面已滚动/交互加载了更多资源），点击列表右上角 **重新扫描**。

### ZIP 内结构示例

```
sitezip_example.com_20260101_120000.zip
├── example.com/
│   ├── index.html
│   ├── images/logo.png
│   ├── minify/app.js
│   └── data/poster/cover.jpg
└── _rendered/
    └── example.com.html   （渲染后的 HTML）
```

## 三、小贴士

- 抓取结果取决于页面已加载的资源；**懒加载**资源需先触发加载（滚动/交互）后再抓取。
- 跨域且未开启 CORS 的资源在降级 fetch 阶段可能无法抓取，但资源树阶段通常已覆盖。
- 单页应用（SPA）建议等待主要内容渲染完成后再打包。
