# SiteZip 技能文档 (Skills)

本文件归档项目涉及的关键技术技能与实现要点，便于二次开发与学习。

## 技能一：Chrome DevTools Protocol 资源抓取

**目标**：以编程方式获取「源代码/来源」面板的资源树。

**要点**：
- 通过 `chrome.debugger.attach({tabId}, "1.3")` 建立 CDP 会话。
- `Page.enable` 后调用 `Page.getResourceTree` 获取 `frameTree`。
- 递归遍历帧树：每个节点包含 `frame`（含 `id`、`url`）与 `resources[]`、`childFrames[]`。
- `Page.getResourceContent({frameId, url})` 返回 `{content, base64Encoded}`。
- 完成后务必 `chrome.debugger.detach` 释放会话，否则调试提示条不消失。

**易错点**：
- `sendCommand` 需用 Promise 包装并检查 `chrome.runtime.lastError`。
- 部分资源（如 opaque / 已失效）取内容会抛错，需 try/catch 逐个跳过。

## 技能二：MV3 内使用 JSZip 打包并下载

**要点**：
- MV3 禁止加载远程脚本，JSZip 必须作为**本地文件**随扩展打包（`lib/jszip.min.js`）。
- 在 **popup 页面**（拥有 DOM 与 `URL.createObjectURL`）中执行打包，避免 service worker 的 API 限制。
- `zip.file(path, content, {base64})` 支持文本与 base64 两种内容。
- `zip.generateAsync({type:"blob", compression:"DEFLATE"}, onProgress)` 支持进度回调。
- `chrome.downloads.download({url: objectURL, filename})` 触发下载。

## 技能三：目录结构还原算法

**要点**：
- 以 URL 的 `hostname + pathname` 作为 zip 内路径，天然还原层级。
- 路径以 `/` 结尾 → 追加 `index.html`。
- 带查询串 → 扩展名前插入短哈希，避免同名覆盖。
- 过滤 `\ : * ? " < > |` 等非法文件名字符。

## 技能四：脚本注入解析 DOM

**要点**：
- `chrome.scripting.executeScript({target:{tabId}, func})` 在页面上下文执行。
- 使用 `new URL(u, document.baseURI).href` 将相对地址转为绝对地址。
- 覆盖 `img/srcset/script/link/video/audio/source/track` 与内联 `style` 的 `url()`。
- 返回值需可结构化克隆（返回普通对象/数组）。

## 技能五：浏览器兼容与降级

**要点**：
- Chrome/Edge 支持 `chrome.debugger`；Firefox 不支持，需运行时检测并降级为 DOM+fetch。
- `manifest.json` 通过 `browser_specific_settings.gecko` 兼容 Firefox。
- 功能检测优先于浏览器 UA 判断（`if (chrome.debugger) {...} else {...}`）。

## 技能六：React 落地页

**要点**：
- React 19 + Tailwind + shadcn/ui + framer-motion 构建高对比暗色落地页
- 入场动效使用 framer-motion `whileInView` + 错峰 `delay` 营造层次。

## 技能七：Swiss / 高对比视觉体系

**要点**：
- 暗色实色背景（不用渐变糊化）、1px 描边替代柔和阴影。
- 强调色单点使用（琥珀黄 `#EAB308`），营造克制而有力的对比。
- 字体分层：展示字体（Cabinet Grotesk）+ 正文（IBM Plex Sans）+ 等宽（JetBrains Mono）。
- Bento 网格 + 大留白 + 非对称栅格。
